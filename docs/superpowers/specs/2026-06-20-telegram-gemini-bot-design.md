# Telegram-бот на Gemini Pro — дизайн

Дата: 2026-06-20

## Цель

Telegram-бот, который обрабатывает входящие сообщения через AI Google Gemini Pro
и отвечает пользователю. Бот ведёт связный диалог (помнит контекст по каждому
пользователю) и имеет одно специальное правило для пятничных поздравлений.

## Требования

1. Приём сообщений из Telegram, ответ в тот же чат.
2. Генерация ответов через Gemini Pro (`google-generativeai`).
3. Память диалога по `chat_id` в ОЗУ (теряется при перезапуске — допустимо).
4. Универсальный ассистент; system prompt и модель настраиваются через `.env`.
5. Специальное правило: на пятничные поздравления («Juma muborak»,
   «Жума муборак», «Juma muborak bo'lsin» и варианты) бот отвечает фиксированной
   фразой `Assalomu alaykum. Rahmat, birgalikda bo'lsin☪️`, минуя вызов AI.
   Этот обмен НЕ записывается в историю диалога.

## Нефункциональные

- Python 3.11+, асинхронный `python-telegram-bot` v21+.
- Минимум зависимостей (подход A — без БД).
- Понятная обработка ошибок и логирование.

### Модули

**config.py** — читает `.env` через `python-dotenv`. Поля:
- `TELEGRAM_BOT_TOKEN` (обязательно)
- `GEMINI_API_KEY` (обязательно)
- `GEMINI_MODEL` (по умолчанию `gemini-2.5-pro`)
- `SYSTEM_PROMPT` (по умолчанию — текст универсального ассистента)
- `MAX_HISTORY` (по умолчанию 20 сообщений)

При отсутствии обязательного поля — падение на старте с понятным сообщением.

**gemini_client.py** — класс `GeminiClient`. Метод
`generate(history: list[dict], user_message: str) -> str`. Конфигурирует
`google-generativeai` ключом, применяет `system_instruction`, отправляет историю
+ новое сообщение, возвращает текст ответа. Ошибки/таймауты пробрасывает
как исключение для обработки в handlers.

**memory.py** — класс `ConversationStore`. Хранит `dict[int, list[dict]]`
(`chat_id -> сообщения` вида `{"role": "user"|"model", "text": str}`).
Методы: `get(chat_id)`, `append(chat_id, role, text)`, `reset(chat_id)`.
`append` обрезает историю до `MAX_HISTORY` последних сообщений.

**special_replies.py** — функция `match(text: str) -> str | None`. Нормализует
текст (нижний регистр, trim) и проверяет вхождение фраз из константного списка
паттернов пятничных поздравлений (латиница и кириллица). Возвращает
фиксированный ответ или `None`. Список паттернов — модульная константа,
легко расширять.

**handlers.py**:
- `/start` — приветствие.
- `/reset` — очистка истории пользователя через `ConversationStore.reset`.
- `on_message` — основной обработчик текста (см. поток ниже).

**main.py** — загружает конфиг, создаёт `GeminiClient`, `ConversationStore`,
регистрирует обработчики, запускает polling.

## Поток обработки сообщения

1. Текст пользователя → `on_message`.
2. `special_replies.match(text)`:
   - если совпало → отправить фиксированный ответ, **выйти** (без AI, без записи
     в историю).
3. Иначе:
   - `history = store.get(chat_id)`
   - `reply = gemini_client.generate(history, text)`
   - отправить `reply` пользователю
   - `store.append(chat_id, "user", text)` и `store.append(chat_id, "model", reply)`
4. При исключении от Gemini — залогировать и отправить пользователю
   «Извините, не удалось обработать запрос, попробуйте ещё раз».

## Обработка ошибок

- Отсутствие токена/ключа на старте → немедленное падение с понятным текстом.
- Ошибка Gemini (сеть, лимиты, таймаут) → лог + дружелюбное сообщение
  пользователю, история не повреждается.

## Тестирование

- `test_special_replies.py`: пятничные фразы (разные варианты, регистр,
  латиница/кириллица) → фиксированный ответ; обычный текст → `None`.
- `test_memory.py`: `append`/`get`, обрезка до `MAX_HISTORY`, `reset`.
- Gemini и Telegram мокаются; внешние вызовы в тестах не выполняются.

## Вне области (YAGNI)

- Персистентное хранилище (SQLite) — можно добавить позже заменой
  `ConversationStore`.
- Голос/изображения, inline-режим, групповая логика, вебхуки (используем polling).
